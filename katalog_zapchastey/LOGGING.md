# Логирование для диагностики проблем с подключением к 1С

В приложение добавлено подробное логирование всех запросов к 1С. Это поможет выяснить причину ошибок на реальном устройстве.

## Как просмотреть логи

### На Android (через adb)

1. Подключите телефон по USB и включите отладку по USB.
2. Откройте терминал и выполните:
   ```bash
   adb logcat | grep -i "OData"
   ```
   Или для более подробного вывода:
   ```bash
   adb logcat | grep -E "(OData|flutter)"
   ```

3. Запустите приложение на телефоне и попробуйте загрузить данные.
4. В логах вы увидите:
   - `OData INFO:` — информационные сообщения (начало загрузки, количество элементов)
   - `OData REQUEST:` — каждый HTTP запрос (метод, URL)
   - `OData RESPONSE:` — ответы сервера (статус код, время выполнения, размер ответа)
   - `OData ERROR:` — ошибки с деталями и стеком вызовов
   - `OData WARNING:` — предупреждения (например, использование localhost)

### В Android Studio / VS Code

1. Откройте вкладку "Logcat" (Android Studio) или "Debug Console" (VS Code).
2. Отфильтруйте по тегу `OData` или уровню `INFO`/`ERROR`.
3. Запустите приложение и наблюдайте логи в реальном времени.

### Примеры логов

**Успешная загрузка:**
```
OData INFO: Загрузка конфигурации из хранилища
OData INFO: Конфиг загружен: baseUrl=http://..., username=Администратор
OData INFO: === Начало загрузки данных каталога ===
OData REQUEST: GET http://.../Catalog_Номенклатура?$format=json&...
OData RESPONSE: 200 | 1234ms | 45678 bytes | http://...
OData INFO: Загрузка завершена за 2345ms: 123 позиций, 456 цен, 789 остатков
```

**Ошибка сети:**
```
OData REQUEST: GET http://172.22.0.62/.../odata/standard.odata/...
OData ERROR in _performGet: SocketException: Failed host lookup: '172.22.0.62'
OData ERROR in loadKronshteinyTree: SocketException: Failed host lookup...
```

**Таймаут:**
```
OData REQUEST: GET http://.../...
OData ERROR in _performGet: Timeout после 60 секунд
OData ERROR in loadKronshteinyTree: Exception: Превышено время ожидания...
```

## Что проверить по логам

1. **URL запросов** — убедитесь, что это не `localhost` или `127.0.0.1` (на телефоне это не сработает).
2. **Время выполнения** — если запросы долгие (>10 секунд), возможно проблема с сетью или сервером.
3. **Статус коды** — 200 = OK, 401/403 = неверный логин/пароль, 404 = неверный URL, 500 = ошибка сервера.
4. **Тип ошибки** — SocketException = нет связи, Timeout = долгий ответ, HTTP 4xx/5xx = ошибка сервера.

## Типичные проблемы и решения

- **"Failed host lookup"** → Проверьте интернет на телефоне и что URL доступен (не localhost).
- **"Timeout"** → Сервер долго не отвечает. Проверьте доступность 1С с телефона (откройте URL в браузере).
- **"HTTP 401/403"** → Неверный логин или пароль 1С. Проверьте настройки.
- **"Operation not permitted"** → Android блокирует HTTP. Используйте HTTPS или прокси с HTTPS.
